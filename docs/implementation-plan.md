## 开发规范

### 每次修改前
1. 查阅本文件确认当前阶段和下一步计划
2. 在 `dev-logs/` 中创建当日日志（文件名 `YYYY-MM-DD.md`）
3. 向用户简要说明要改什么、为什么改、影响范围
4. 获得用户确认后再动手

### 每次修改后
1. 在桌面浏览器中验证 21 个步骤均可正常走通
2. 更新 `dev-logs/` 当日日志，标记完成事项
3. 如涉及规范变更，同步更新 `docs/` 中对应文档
4. 修改稳定后 `git push` 更新线上版本

### 不做的事
- 不引入外部依赖（npm 包、CDN 资源）
- 不添加超出当前阶段计划的功能
- 不修改与当前任务无关的代码
- 不在没有文档更新的情况下做架构性改动
- 不同时修改多个独立功能
