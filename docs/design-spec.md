# 设计规范文档

## 设计原则

### 核心设计理念

为智力障碍儿童设计，遵循以下原则：

1. **极简至上**：每屏只有一个操作目标，消除一切干扰
2. **高对比度**：操作目标与背景有明确视觉区分
3. **实物风格**：使用实物照片风格的视觉表现（PNG 图片素材）
4. **零文字依赖**：所有引导通过语音+大图完成
5. **宽容交互**：不要求精确操作，拖拽重叠 35% 即通过
6. **正向激励**：只有鼓励和庆祝，绝不出现负面反馈

## 色彩系统

### 主色调

| 颜色名 | 色值 | 用途 |
|--------|------|------|
| 背景米色 | `#FFF5EB` | 全局背景，温暖柔和 |
| 卡片白 | `#FFFFFF` | 卡片和交互区域背景 |
| 主色橙 | `#FF8C42` | 按钮、进度点、箭头、强调元素 |
| 深橙 | `#FF6B00` | 拖放区发光框、标签 |
| 主色蓝 | `#5BA0D5` | 第二关卡卡片、水相关元素 |
| 成功绿 | `#6BCB77` | 步骤完成、进度点完成 |
| 温柔粉 | `#FF7B9C` | 第三关卡卡片、毛巾 |

### 文字颜色

| 颜色名 | 色值 | 用途 |
|--------|------|------|
| 深棕 | `#3A2A1A` | 标题文字 |
| 浅棕 | `#8B7355` | 辅助文字 |
| 标签橙 | `#E05500` | 箭头标签文字 |

### 色彩使用规则

- 背景只用暖色系（米色/奶油色渐变）
- 操作目标使用饱和色，与背景拉开对比
- 成功状态使用绿色
- 不使用红色（避免负面联想）
- 拖放目标区使用橙色发光框 + 半透明橙色底色

## 排版规范

### 字体

```css
font-family: -apple-system, "PingFang SC", "Noto Sans SC", "Microsoft YaHei", sans-serif;
```

优先使用系统字体，确保在 iPad 上渲染清晰。

### 字号（使用 clamp() 响应式）

| 级别 | 字号范围 | 用途 |
|------|---------|------|
| XL | 32px - 48px | 首页标题 |
| LG | 24px - 36px | 关卡卡片标题、步骤引导语 |
| MD | 18px - 24px | 庆祝标题 |
| SM | 14px - 18px | 辅助说明文字 |

### 箭头标签

- 字号：16px
- 字重：700
- 颜色：`#E05500`
- 背景：白色 96% 不透明，圆角药片形状
- 阴影：橙色投影 + 微阴影

## 触控交互设计

### 触控目标尺寸

- 所有可交互元素最小尺寸：**44×44pt**
- 关卡卡片：宽 200-280px
- 拖拽物品：尺寸适中（如牙刷 180×130px，手部 72×72px）
- 箭头指引：96×96px SVG + 标签

### 手势触发阈值

| 手势 | 触发条件 |
|------|---------|
| 点击 | 移动 < 18px |
| 拖拽成功 | 物品与目标重叠 > 35% |

### 箭头指引

每种拖拽操作都有可见的动态箭头提示：
- 箭头统一使用 **96×96px SVG**，内含白色圆形底托 + 橙色箭头图形
- 底托样式：白色 88% 不透明 + 橙色细描边 + 外光晕
- 动画：沿操作方向来回移动 30px + 1.06 倍缩放呼吸（1.2s 循环）
- 文字标签在箭头下方，白色药片形状

### 拖放目标可视化

- 所有 `.drag-target` 用橙色虚线框（5px、脉动动画）标示
- 透明不可见的目标区加半透明橙色底色 + 实线边框
- 重点步骤（如刷牙对口部）加发光拖放区（`.drop-zone`）+ "刷这里"标签

### 交互反馈时机

```
用户操作
    ↓
手势检测（即时）
    ↓
┌─ 成功 ─────────────────────────┐
│  1. 交互区域绿色描边（立即）     │
│  2. 成功三连音（立即）           │
│  3. 随机表扬语音（立即）         │
│  4. 星星粒子动画（立即）         │
│  5. 进度点更新（立即）           │
│  6. 等待 2 秒                   │
│  7. 步骤切换动画（0.25s 出 + 0.45s 入）│
│  8. 新步骤手势启用               │
└────────────────────────────────┘
┌─ 失败 ─────────────────────────┐
│  1. 交互区域抖动（立即）         │
│  2. 柔和低音（立即）            │
│  3. 随机鼓励语音（立即）         │
│  4. 拖拽物品弹回原位（0.4s）     │
│  5. 等待 0.6s 恢复              │
└────────────────────────────────┘
```

## 场景设计

### 浴室场景（刷牙 + 洗脸关卡）

- 使用 `bathroom-bg.png` 照片级浴室背景
- 场景上方叠加步骤物品（牙刷/牙膏/水杯/毛巾/水龙头等 PNG 素材）
- 人物照片（`brush-boy.png` / `cartoon-boy.png`）用于刷牙和擦脸步骤

### 卧室场景（穿衣关卡）

- CSS 渐变暖色背景：`#FFF3E8 → #FFECD8 → #FFE4CC`
- 衣物使用 SVG 绘制（橙色外套 + 白色领口 + 袖子 + 拉链）
- 手部拖拽使用 `hand.png`

### 物品素材

| 物品 | 格式 | 文件名 |
|------|------|--------|
| 牙刷 | PNG | `toothbrush.png` |
| 牙膏 | PNG | `toothpaste.png` |
| 水杯 | PNG | `cup.png` |
| 毛巾 | PNG | `towel.png` |
| 手部 | PNG | `hand.png` |
| 洗漱台水龙头 | PNG | `sink-faucet.png` |

## 动画规范

### 动画时长

| 动画 | 时长 | 缓动 |
|------|------|------|
| 步骤进入 (stageIn) | 450ms | ease-out |
| 步骤退出 (stageOut) | 250ms | ease-in |
| 弹跳（标题图标） | 2000ms | ease-in-out 循环 |
| 箭头滑移 | 1200ms | ease-in-out 循环 |
| 点击脉动 (tapPulse) | 1000ms | ease-in-out 循环 |
| 目标框脉动 (targetPulse) | 1000ms | ease-in-out 循环 |
| 拖放区发光 (zoneGlow) | 900ms | ease-in-out 循环 |
| 错误抖动 (shake) | 500ms | ease |
| 拖拽弹回 (snap-back) | 400ms | ease-out |
| 星星粒子上升 | 1000-2500ms | ease-out |
| 撒花下落 | 2000-5000ms | ease-in |
| 步骤成功后等待 | 2000ms | — |

### 动画使用规则

- 不使用快速闪烁（避免诱发不适）
- 不使用大面积快速移动
- 所有循环动画都有足够的静止帧
- 粒子数量限制（每次 ≤ 8 个星星，≤ 50 个彩纸）

## 音效设计

### 语音规范

- 语言：中文普通话（zh-CN）
- 语速：0.75x（比正常略慢，便于儿童理解）
- 音高：1.0x（自然女声）
- 音量：0.95（足够清晰）
- 优先选择：Tingting → Sinji → Meijia → Lili → 任意 Mandarin 女声

### 音效规范

| 音效 | 频率 | 时长 | 波形 | 含义 |
|------|------|------|------|------|
| 成功音 | 523→659→784Hz | 每音 140ms/280ms | sine | 步骤完成 |
| 失败音 | 220Hz | 250ms | triangle | 再试一次 |
| 庆祝音 | 523→1047Hz 音阶 | 每音 180ms | sine | 关卡完成 |

### 音效使用规则

- 所有音效柔和、短促
- 不使用尖锐/刺耳频率
- 不叠加过多音效
- 提供一键静音功能

---

## 无障碍设计

- 高对比度模式（通过颜色选择保证）
- 大触控目标（≥ 44pt）
- 语音引导替代文字
- 箭头动画指引操作方向
- 目标区域可视化（虚线框 + 发光框）
- 禁止意外缩放
- 禁止意外选择
